Welcome to Smartmap

For the design challenge(proftaak) we(our proftaak group consisting of 6 persons) had a brainstorm session where we had to come up with 150 possible problems that we would like to solve, at the end we had to choose one problem. Our problem was with the public paper maps at bus stops being unclear. They show you all the lines instead of the line you're interested in.
After we described our problem we had to come up with a solution for this problem. Our solution is an interactive public transport map application called smartmap. We want to place touchscreens at every bus stop where you can interact with the map where you can search for a bus line or select pre defined lines, all possible stops and information about each stop such as the timestamps.

This project integrates five different aspects of our study that we've followed in the first 7 weeks. These are:

  • ICT & Technology
  • ICT & Software Engineering
  • ICT & Infrastructure
  • ICT & Business
  • ICT & Media

API Documentation

Method Possible Routes Description
GET /journey/ This will return a list of all journeykeys of busses that are currently driving in Noord-Brabant. This is specified by the Data Owner Code CXX
GET /journey/eindhoven This will return a list of journeykeys of busses that are currently driving in Eindhoven.
GET /actuals/eindhoven/:journeykey This will return a list of the actual information for that particular bus. You will need the journey key from the previous routes to access this data an example of a possible actual route would be /actuals/eindhoven/400
GET /lines This will return a list of driving bussess throughout the entire country
GET /lines/dacs/ This will return a list of area owner codes. These are needed to access data per region. ARR is arriva and CXX is Bravo
GET /route This will return a list of all coordinates for each line that represent a bus stop. We use these coordinates to build the directions line from start destination to end destination
GET /route/:line This will return a list of all coordinates for a specific line that represent a bus stop. We use these coordinates to build the directions line from start destination to end destination.
NOTE: You will need to add a capital L before the linenumber otherwise you won't get any data back. possible lines: [L400, L401, L402, L403, L404, L405, L406, L407]
GET /bus/stop Returns the stopcode for a bus, value will be either 0 or 1
POST /bus This route is used to set the stopcode for a bus. When status is 1 a red LED will be turned on to notify the bus driver

How it works

We have built an interactive web based application that consumes our API data to make it interactive. On launch the default map will be shown and you will have a list selection of 7 possible lines. Each line representing the direction and stops for that bus. When clicking on one stop, you will be able to find all information that belongs to that stop.
